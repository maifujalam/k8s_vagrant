1. It uses runtime class to select container runtime.
2. There are many container runtimes:- docker,podman,crio etc.
3. Selecting which container runtime pod should use,we can select it creating runtimeclass and use it inside pods specification.
    apiVersion: v1
    kind: Pod
    metadata:
      name: mypod
    spec:
      runtimeClassName: myclass

4. runtime class have 2 things:
    a. runtime class name.
    b. handler name.
5. For Gvisor runtime class is:-
    apiVersion: node.k8s.io/v1
    kind: RuntimeClass
    metadata:.
      name: gvisor
    # The name of the corresponding CRI configuration
    handler: runsc
6. Verify if container is using gvisor: $ k exec -it sec -- dmesg
